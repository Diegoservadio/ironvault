01 — Il problema

Le PMI italiane
nel mirino

Il ransomware è la minaccia numero uno per il tessuto produttivo italiano. La grande maggioranza delle vittime non ha strumenti di protezione adeguati.

78%
degli attacchi ransomware colpisce PMI — spesso senza IT interno
+65%
crescita attacchi gravi in Italia nel 2023 rispetto all'anno precedente
94%
degli attacchi parte da una email di phishing — il vettore principale
20 k€
costo medio di un attacco ransomware per una piccola impresa italiana
Dal 1 gennaio 2025 la Direttiva NIS2 (D.Lgs. 138/2024) impone obblighi di backup e continuità operativa anche alle PMI. Sanzioni fino al 2% del fatturato globale per inadempienza.
02 — La soluzione

Architettura IronVault

Infrastruttura ibrida on-premises e cloud con backup immutabile e recovery automatico. Costruita interamente con tecnologie open source e AWS Free Tier.

On-premises — VMware
Infrastruttura locale simulata
  • Ubuntu Server 22.04 — file server + backup agent + stack Grafana SOC
  • Windows Server 2022 — Active Directory, GPO IronVault-Security-Policy
  • Windows 10 — postazione Laura Bianchi, punto di ingresso dell'attacco
  • Apache2 — portale intranet Studio Contabilità Milano su porta 80
  • Backup automatico ogni ora tramite cron job su S3
AWS Cloud — eu-north-1
Layer di protezione cloud
  • S3 Object Lock COMPLIANCE 30gg — backup WORM assolutamente inviolabile
  • IAM Least Privilege — DENY esplicito su DeleteObject per backup agent
  • Bucket Policy — DENY assoluto delete per qualsiasi Principal, incluso admin
  • EC2 t3.micro standby — recovery completo in ~2 minuti
  • CloudWatch + SNS — alert automatico su accessi non autorizzati
Protezioni S3 a strati
🔒

Object Lock COMPLIANCE

Nessuno può cancellare i backup per 30 giorni — nemmeno l'account root AWS

🚫

IAM DENY esplicito

Il backup agent non può eseguire DeleteObject — prevale su qualsiasi Allow futuro

🛡️

Bucket Policy DENY

Blocca il delete da qualsiasi Principal — incluso l'amministratore dalla console AWS

📋

Versioning + Lifecycle

VersionId unico per ogni backup. Glacier dopo 30gg, eliminazione automatica dopo 1 anno

03 — La demo

Attacco → Recovery
in ~2 minuti

Sequenza completamente automatica e dimostrabile live. Da un singolo doppio click al ripristino integrale su EC2.

01

Email phishing — il vettore

Laura apre Outlook Web e vede una email con grafica ufficiale dell'Agenzia delle Entrate. Scarica il PDF allegato nella cartella Downloads.

vettore: phishing
02

Esecuzione ransomware automatica

Doppio click sul dropper mascherato da PDF. F: G: H: si cifrano, il trigger server riceve il segnale HTTP e lancia lo script su Ubuntu. Il sito intranet mostra la pagina di compromissione.

F: G: H: → .locked | intranet offline
03

Grafana rileva l'attacco

La dashboard SOC mostra il picco di scrittura su disco. THREAT LEVEL passa da SECURE a UNDER ATTACK. L'email di alert IronVault arriva con dettagli e comandi di recovery.

alert email automatico
04

Air-gap S3 — backup inviolabile

aws s3 rm → AccessDenied — explicit deny
Anche con le credenziali rubate dal server, il backup è intoccabile.

IAM deny dimostrato live
05

Recovery su EC2 — RTO misurato con timer

Avvio istanza → SSH → time bash recovery.sh → 144 file ripristinati in ~3 secondi. Il sito intranet torna online. RTO totale: ~2 minuti.
